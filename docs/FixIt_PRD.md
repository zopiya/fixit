# FixIt 产品需求文档 (PRD)

**版本：** V1.0  
**状态：** 定稿  
**最后更新：** 2026-06-06

---

## 一、产品概述

### 1.1 产品定位

**FixIt** 是一款面向前端开发工程师的 Chrome 浏览器扩展。它解决开发者在预览网页时，将肉眼发现的 UI 问题高效转化为 AI 可直接执行的结构化工单这一核心痛点。

核心理念：**轻量、无感、去中心化**。完全依附浏览器原生生态，不依赖云端服务器，保障开发者源码隐私。

### 1.2 目标用户

**主要用户：** 使用 AI 编码工具（Claude Code、OpenCode、Cursor、Windsurf 等）进行日常开发的前端工程师。

**用户场景：** 在本地开发环境（`localhost:3000` 等）或线上预览页面时，发现 UI 细节问题，需要将问题精确描述给 AI Agent 执行修改。

### 1.3 核心痛点

当前开发者的典型工作流存在严重的"翻译损耗"：

```
看到问题 → 心里有思路 → 用文字描述问题 → AI 理解偏差 → 反复沟通对齐
```

这个"翻译"环节耗时且容易失真。开发者必须用文字描述"第三屏左侧导航栏的第二个按钮"，AI 还不一定能准确定位。

**FixIt 的价值：** 用点选代替描述，用精确 CSS Selector + XPath 双轨定位替代模糊语言，将翻译损耗压到接近零。

## 二、功能需求

### 2.1 核心功能（V1 范围）

#### 功能一：快捷键唤醒与激活

- 快捷键 `Alt + F` 唤醒/关闭 FixIt 标注模式。
- 点击浏览器工具栏插件图标同样可以切换激活状态。
- 激活后，插件图标视觉状态变化（高亮/激活色），给用户明确的状态反馈。
- 激活态在标签页切换后保持各自独立的状态。

#### 功能二：悬停高亮与元素点选

- 激活后，鼠标在页面上移动时，悬停元素出现高亮边框（颜色：品牌主色，2px 实线，无侵入感）。
- 高亮框通过 Shadow DOM 渲染，**绝不污染目标网页的任何 CSS 样式**。
- 点击目标元素后，原地弹出批注输入框（同样基于 Shadow DOM 隔离）。

#### 功能三：批注输入与确认

- 批注输入框为单行/多行文本输入，支持 `Enter` 确认、`Esc` 取消。
- 确认后，目标元素原地出现一个精致的数字序列徽章（①②③…），徽章样式通过 Shadow DOM 渲染。
- 徽章可悬停查看批注内容摘要，点击可编辑或删除该条标注。
- 同一页面支持累计多条标注，理论上不限数量。

#### 功能四：双轨元素定位

每次点选确认后，后台自动运行双轨定位算法，生成两个定位标识并存储：

**主轨：智能语义 CSS Selector**，按以下优先级依次尝试，找到第一个在当前文档内唯一命中目标元素的选择器即停止：

| 优先级 | 定位依据 | 示例 |
|--------|----------|------|
| 1 | `data-testid` / `data-cy` / `data-qa` | `[data-testid="submit-btn"]` |
| 2 | 有语义的 `id`（排除含哈希特征的） | `#login-form` |
| 3 | `aria-label` / `role` 组合 | `button[aria-label="关闭"]` |
| 4 | 表单元素 `name` 属性 | `input[name="email"]` |
| 5 | 语义标签 + 有意义 class（过滤 hash class） | `.btn-primary` |
| 6 | 向上爬 3 层 DOM 的结构路径 | `#login-form > .actions > button:last-child` |

每一级生成候选选择器后，立刻用 `document.querySelectorAll()` 验证唯一性，唯一则采用。同时记录 `cssSelectorConfidence` 置信度级别，在工单中标注给 AI 参考。

**副轨：优化截断版 XPath**，向上遍历 DOM 树，从最近的有稳定属性（`id` 或 `data-*`）的祖先节点开始生成相对 XPath，而非从根节点 `/html` 硬走绝对路径。

```
// 脆弱的绝对路径（不用）
/html/body/div[3]/div[1]/form/div[2]/button[1]

// 优化截断版（采用）
//form[@id='login-form']//button[contains(@class,'submit')]
```

#### 功能五：本地持久化存储

- 基于 `chrome.storage.local` 实现本地持久化，刷新页面、重启浏览器后标注不丢失。
- 存储 key 使用 `origin + pathname`（忽略 query string 和 hash），同一路径的不同参数状态归为一组。
- 完整 URL 作为辅助字段单独保留备用。
- 不产生任何网络请求，完全本地运行。

#### 功能六：侧边栏看板（Side Panel）

- 通过 Chrome Side Panel API 实现右侧侧边栏面板。
- 侧边栏内容与当前激活标签页绑定，切换标签页时自动刷新显示对应页面的标注列表。
- 标注列表按创建顺序排列，每条展示：序号徽章、元素定位摘要、用户批注内容、删除按钮。
- 支持单条标注删除与全部清空。
- 底部固定"**复制 AI 工单**"按钮，点击后将结构化 Markdown 工单复制到剪贴板，并显示 2 秒成功提示。

#### 功能七：结构化 Markdown 工单导出

生成的工单遵循面向 AI Agent 优化的固定模板，确保 AI（Claude Code、OpenCode 等）能直接解析并定位源码执行修改。

**工单模板：**

```markdown
# FixIt 修改工单
页面：{origin + pathname}
导出时间：{timestamp}

---

## ① {用户批注内容摘要}
**修改要求：** {用户完整批注}

**元素定位：**
- CSS Selector：`{cssSelector}`
- 定位置信度：{confidence}（高/中/低可靠）
- XPath 备用：`{xpath}`

**元素快照：**
```html
{htmlSnapshot}
```

---

## ② ...（以此类推）
```

### 2.2 非功能需求

- **零配置：** 安装即用，无需登录、无需 API Key、无需任何初始设置。
- **零网络：** 不发出任何网络请求，源码和批注数据完全留在本地。
- **性能：** 插件激活响应时间 < 100ms，侧边栏打开 < 200ms，标注写入 < 50ms。
- **兼容性：** 支持 `localhost` 开发环境和任意线上网页，行为完全一致。
- **无侵入：** 所有 UI 元素（高亮框、批注气泡、数字徽章）通过 Shadow DOM 渲染，不影响目标网页任何功能或样式。

### 2.3 明确不做（V1 排除项）

- 不引入云端同步功能。
- 不集成任何 AI API（V1 只做原始工单生成，AI 润色为后续规划）。
- 不做视觉微调滑块（V2 规划）。
- 不做团队协作与标注共享。
- 不支持 Firefox 或其他浏览器。

## 三、用户流程

### 3.1 标准使用 SOP

```
[快捷键唤醒] → [点选元素] → [写批注] → [继续标记] → [侧边栏复制工单] → [投喂 AI Agent]
```

**第一步（唤醒）：** 在浏览器中打开正在开发的网页，按下 `Alt + F` 激活 FixIt。

**第二步（点选）：** 鼠标移动到不满意的元素，点击，原地弹出批注输入框，输入修改要求（如："这个按钮换成蓝色，加 hover 动画"），按 `Enter` 确认。元素原地出现 ① 标签。

**第三步（继续）：** 滚动页面，继续点选问题元素，依次生成 ②③ 标签。

**第四步（导出）：** 打开右侧 Side Panel，所有标注整齐列出，点击底部"**复制 AI 工单**"。

**第五步（落实）：** 切换到 Claude Code / OpenCode / Cursor，`Ctrl + V` 发送，AI Agent 开始执行修改。

### 3.2 页面切换行为

- 切换到新标签页，侧边栏自动切换显示该标签页的标注列表。
- 同一标签页刷新后，页面上的数字徽章会重新注入渲染，标注数据不丢失。
- 不同 URL 路径的标注完全隔离，互不干扰。

## 四、新手引导设计

### 4.1 设计原则

开发者最讨厌冗长说明书和强制弹窗。FixIt 采用"**1分钟线上游乐场**"的引导模式，让用户在无压力环境中完成真实操作，建立肌肉记忆。

### 4.2 触达机制

用户首次安装完成后，插件自动在新标签页打开内置游乐场页面：`chrome-extension://.../playground.html`。

### 4.3 游乐场内容

游乐场是一个**故意设计得有明显问题**的模拟网页，顶部醒目提示：

> 💡 **欢迎来到 FixIt 游乐场！**
> 我们为你准备了一个"故意写得很烂"的页面。请按下 `Alt + F` 激活 FixIt，然后试着修理它。

页面侧边悬浮三步任务引导：

1. **🎯 任务一：** 点击【红色提交按钮】，添加批注："帮我换成漂亮的 Tailwind 蓝色，加个 hover 放大动画"
2. **📝 任务二：** 点击【错位的标题】，添加批注："移动端居中，大屏左对齐"
3. **🚀 任务三：** 展开右侧侧边栏，点击复制，查看生成的 AI 工单

**通关反馈：** 完成三步操作并成功复制后，页面绽放轻量级烟花动画，并提示：
> "恭喜！你已掌握 FixIt 的全部诀窍。现在，去指点你的本地网页吧！"

### 4.4 三无原则

1. **无登录注册：** 安装即用，不弹出任何注册/登录阻断流程。
2. **无 API Key：** 纯本地运行，不要求用户配置任何 Token 或付费账户。
3. **无环境依赖：** `localhost` 与线上网页体验完全一致。

## 五、阶段规划

### V1 阶段：核心闭环（当前聚焦）

**目标：** 实现从"网页点选标注"到"一键复制完美 AI 工单"的完整闭环。

**功能范围：** 双轨定位算法、Shadow DOM 隔离批注、本地持久化、Side Panel 看板、Markdown 工单导出、Playground 新手引导。

**体验指标：** 零配置、零登录、零网络请求、秒开。

### V2 阶段：AI 润色增强（后续规划）

> 原则：V1 阶段不编写任何 V2 业务代码，但在数据结构上预留字段接口。

在插件内引入 AI 能力，对用户的原始批注进行自动润色，生成更精准、更适合 AI Agent 解析的修改指令，进一步提升 Agent 一次性执行成功率。

### V3 阶段：视觉深度辅助（远期规划）

引入轻量级视觉微调滑块（间距、圆角、颜色预设），将用户微调后的精确 CSS 差值作为补充上下文注入 Prompt。具体形态在 V1 收集用户反馈后定夺。
